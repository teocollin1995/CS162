AdjacencyMatrixChecker

Proposed by Joey Zou on 01/07/15
Last edited on 01/07/15

A graph is comprised of a set of vertices and a set of edges, where an edge is a pair of vertices. A simple undirected graph is a graph where there are no loops (i.e. an edge from a vertex to itself), the order of the pair of vertices of an edge is not important (so (a,b) is an edge if and only if (b,a) is an edge, and both pairs are considered one edge), and there is at most one edge for every pair of vertices. The adjacency matrix of a graph is a v by v matrix (where v is the number of vertices) where the entry of the ith row and jth column is 1 if there is an edge between vertices i and j and 0 otherwise. There are four necessary and sufficient conditions for a matrix to be the adjacency matrix of some simple undirected graph:

  • The matrix must be square
  • Each entry of the matrix is either 0 or 1
  • The entries along the diagonal are all 0
  • The matrix is symmetric, i.e. interchanging the roles of the rows and columns does not change the matrix

Problem: Write a program which, given a matrix, determines whether the matrix is the adjacency matrix of some simple undirected graph. Additionally, if possible, print a graphic representation of the graph. You may represent the matrix using whatever data structure you like.

A simple take on the problem in C (by Nick Sapoval)

/* This is a simple adjacency matrix checker.
No command line parameters needed. The
interaction looks like follows:

$ ./matrix
3
0 1 1
1 0 0
1 0 0
This is an adjacency matrix.

$ ./matrix
3
1 2 3
This is not an adjacency matrix.
*/

#include <stdio.h>
#include <stdlib.h>

int main(int argc, char **argv) {
    int n;
    scanf("%d", &n);
    int a[n][n];
    int v=0,i,j;
    for(i=0; i<n; i++) {
        for(j=0; j<n; j++) {
            scanf("%d", &v);
            if(v==0 || v==1) a[i][j]=v;
            else {v=-1; break;}
            if(i>j && a[i][j]!=a[j][i]) {v=-1; break;}  
            if(i==j && a[i][j]!=0) {v=-1; break;}
        }
    }
    printf("This is %s", v == -1 ? "not an adjacency matrix." : "an adjacency matrix.");
    exit(0);
}

Another solution in Python (by Sotiri Komissopoulos)

#!/usr/bin/env python3.4

"""
A simple command-line python script to determine whether a given matrix is the adjacency matrix of some graph. If not, a reason why is given.
Graphics for the corresponding graph have not been implemented.

Sample runs:
$ echo -e "1 0 1\n0 0 0\n1 0 0" | python3.4 adjacencyMatrixChecker.py
Adjacency matrices must have only zeroes on their diagonal!
1 0 1
0 0 0
1 0 0
is NOT an adjacency matrix!

$ echo -e "0 0 1\n0 0 1\n1 1 0" | python3.4 adjacencyMatrixChecker.py
0 0 1
0 0 1
1 1 0
is an adjacency matrix!
"""

from sys import stdin

def checkAdj(mat):
    dim = len(mat)

    for i in range(dim):
        row = mat[i]

        if row[i] != "0":
            print("Adjacency matrices must have only zeroes on their diagonal!")
            return(False)

        if any(val not in ["0","1"] for val in row):
            print("Adjacency matrices must only contain zeroes and ones!")
            return(False)

        elif len(row) != dim:
            if len(row) > dim:
                print("Adjacency matrices must be square! There are fewer rows than columns.")
            else:
                print("Adjacency matrices must be square! There are more rows than columns.")
            return(False)

        elif not(all(mat[i][j] == mat[j][i] for i in range(dim) for j in range(i))):
            print("Adjacency matrices must be symmetric!")
            return(False)

    return(True)



def showMat(mat):
    return("\n".join(list(map(" ".join,mat))))



inpMat = []

for line in stdin:
    inpMat.append(line.split())

if checkAdj(inpMat):
    print(showMat(inpMat))
    print("is an adjacency matrix!")
else:
    print(showMat(inpMat))
    print("is NOT an adjacency matrix!")

A new take on this problem in R by Teo Collin

So, I have decided that the wiki needs some R, so I decided to quickly write this in R. The result in an R function that can take a data frame, generated in R or read in from a csv, and determine if that data frame is a viable adjacency matrix. One will then note that this is intended to be used as part an "R" command line interaction.

I will quickly explain the program.

The first test is conducted using an assumption about data frames, mainly that all vectors in a data frame will be the same length. Then, one could just check the number of vectors in a data frame vs. the length of any given vector. However, there is an exception to this assumption. If a data frame is read in from some external source, all vectors will be filled with NA at the end until they are as long as the longest vector. To deal with this, I remove all rows with NA and then test the number of vectors against the length of the first vector.

The second test loops through all vectors in the data frame and adds the length of the vector with all values of that vector except for 1s and 0s to a constant that starts as 0. Then, we just check if the value of the constant is not 0 for this test.

The third test is conducted in a straightforward manner and can easily be discerned by looking at the code.

The fourth test is conducted by transposing our canmdidate matrix and testing equality between the values at each position of each matrix e.g. (1,1) in the transposed matrix is compared against (1,1) in the original only for all possible positions. (R operators are often vector or data frame wise...). This yields a data frame of boolean vectors, which I loop over using reduce (i.e foldr in haskell), a constant, and a for loop.

I think that there are probably more concise ways of doing this. Anyway, here is the function:

adjacencymatrix <- function(sample){
sample <- na.omit(sample)
test1 <- length(sample) == length(sample[,1])
# is square
temp <- 0
for (x in sample){
temp <- temp + length(x[x != 1 && x !=0])
}
test2 <- temp[1] == 0
#only 0 and 1s

test3 <- TRUE
for (x in seq(1,length(sample))){
        test3 <- test3 && (sample[x,x] == 0)
}
# diagonal only 0

test4 <- TRUE
for(x in (t(sample) == sample)){
test4 <- test4 && Reduce(function(x,y) {x && y}, x, TRUE)
}
#survives being transposed
return (test1 && test2 && test3 && test4)
}

Another solution in MIT Scheme by Teo Collin

In a similar vein, I decided that I wanted to see some scheme on this wiki so I quickly wrote this program in scheme. It is once again just a function. It takes 2d lists as its matrix representation.

I learned scheme from TLS and the GNU/MIT docs. This code is somewhat reflective of this. I also have not written any scheme in a while so it could probably use second pass.

I do not think the code requires much explanation beyond looking at it so I will not waste breath on that. Code:

;;This already exists, but practice
(define length
        (lambda (list)
        (cond ((null? list) 0)
                 (else (+ 1 (length (cdr list)))))))

;;Maps length
 (define mapL
        (lambda (list) (map length list)))
;;What it says on the tin
 (define test1? (lambda (list)
        (fold-right (lambda (x y) (and x y)) #t (map (lambda (x) (eq? x (length list))) (mapL list)))))

;;if all elements in a list are 0 or 1.
 (define zone? (lambda (list)
        (cond ((null? list) #t)
        ((not (or (eq? (car list) 0) (eq? (car list) 1))) #f)
                (else (zone? (cdr list))))))

 (define test2? (lambda (list)
        (cond ((null? list) #t)
        ((not (zone? (car list))) #f)
                (else (test2? (cdr list))))))

 (define test3? (lambda (list)
        (cond ((null? list) #t)
        ((not (zero? (car (car list)))) #f)
                (else (test3? (map cdr (cdr list)))))))

(define (transpose m) (apply map list m))
;;I knew there was an easy one for mit scheme so I just googled itâ€¦ http://rosettacode.org/wiki/Matrix_transposition#Scheme

 (define test4? (lambda (list) (equal? (transpose  list) list)))

 (define amatrix? (lambda (list)
        (and (and (test1? list) (test2? list)) (and (test3? list) (test4? list)))))
;;End function

;;Tests:
(amatrix? (list (list 0 0 1) (list 0 0 1) (list 1 1 0)))

 (amatrix? (list (list 0 0 1) (list 0 0 1)))
;;False
 (amatrix? (list (list 0 0 1) (list 0 0 1) (list 1 1 2)))
;;False
 (amatrix? (list (list 0 0 1) (list 1 0 1) (list 1 1 0)))
;;False
 (amatrix? (list (list 1 0 1) (list 0 0 1) (list 1 1 0)))
