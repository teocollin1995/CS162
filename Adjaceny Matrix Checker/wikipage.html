Notes: The problem gives the necessary background information (or an easy way to access it). The specification is pretty clear in that after reading the problem you know exactly what the program needs to do. What makes this an easier problem is that the 'how' is actually quite clear and the problem can be completed with the basic constructs of almost any programming language. Harder problems should require considerably more.

Problem:

Proposed by Joey Zou on 01/07/15
Last edited on 01/07/15

A graph is comprised of a set of vertices and a set of edges, where an edge is a pair of vertices. A simple undirected graph is a graph where there are no loops (i.e. an edge from a vertex to itself), the order of the pair of vertices of an edge is not important (so (a,b) is an edge if and only if (b,a) is an edge, and both pairs are considered one edge), and there is at most one edge for every pair of vertices. The adjacency matrix of a graph is a v by v matrix (where v is the number of vertices) where the entry of the ith row and jth column is 1 if there is an edge between vertices i and j and 0 otherwise. There are four necessary and sufficient conditions for a matrix to be the adjacency matrix of some simple undirected graph:

  • The matrix must be square
  • Each entry of the matrix is either 0 or 1
  • The entries along the diagonal are all 0
  • The matrix is symmetric, i.e. interchanging the roles of the rows and columns does not change the matrix

Problem: Write a program which, given a matrix, determines whether the matrix is the adjacency matrix of some simple undirected graph. Additionally, if possible, print a graphic representation of the graph. You may represent the matrix using whatever data structure you like.

Solution:

So, I have decided that the wiki needs some R, so I decided to quickly write this in R. The result in an R function that can take a data frame, generated in R or read in from a csv, and determine if that data frame is a viable adjacency matrix. One will then note that this is intended to be used as part an "R" command line interaction.

I will quickly explain the program test by test.

The first test is conducted using an assumption about data frames, mainly that all vectors in a data frame will be the same length. Thus, one could just check the number of vectors in a data frame vs. the length of any given vector. However, there is an exception to this assumption. If a data frame is read in from some external source, all vectors will be filled with NA at the end until they are as long as the longest vector. To deal with this, I remove all rows with NA and then test the number of vectors against the length of the first vector.

The second test just loops through the vectors and checking if each vector has a length other than 0 after all the 0s and 1s have been removed from it.

The third test is conducted in a straightforward manner and can easily be discerned by looking at the code.

The fourth test is conducted by transposing our candidate matrix and testing equality between the values at each position of each matrix e.g. (1,2) in the transposed matrix is compared against (2,1). All of this happens in the line "t(sample) == sample" because the R equality operator can operate on data frames so that it yields a data frame of boolean vectors. I loop over the dataframe of bools to find a FALSE (where the transposed position is not equivalent to the original) using reduce (i.e foldr in haskell), a constant, R's version of lambda (function()), and a for loop.

I think that there are more concise ways of doing this. Anyway, here is the function:

adjacencymatrix <- function(sample){
sample <- na.omit(sample)
test1 <- length(sample) == length(sample[,1])
#test1: is square
if (test1 == FALSE) { return FALSE }

temp <- 0
for (x in sample){
if (length(x[x != 1 && x !=0]) != 0) { return FALSE }
}
#test2: only composed of 0 and 1s

test3 <- TRUE
for (x in seq(1,length(sample))){
        if (sample[x,x] != 0) { return FALSE }
}
#test3: diagonal only 0

test4 <- TRUE
for(x in (t(sample) == sample)){
test4 <- test4 && Reduce(function(x,y) {x && y}, x, TRUE)
}
#survives being transposed
return (test4)
}
