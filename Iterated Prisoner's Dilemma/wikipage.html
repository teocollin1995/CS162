The problem statement provides a lot of background information and is mostly clear on what the requirements, but probably could have been clearer. Also, I kind of just like this problem because it could possibly involve a sort of wiki wide competition although that didn't really happen.

In any event, for creating problems that are not as basic as problem 1 but not as large as problem 3, you have a few options:

  • Make the task sufficiently complex as to require some thought to be spent on the design and a lot of code (e.g. simpler types of neural nets might fit this description
  • Require the the use/mastery of some type of outside library (e.g. web scraping or mass image manipulation)
  • Place some type of efficiency restriction on the problem.
  • Require the mastery/use of a somewhat complex algorithm, data structure, or concept (e.g. implementing a bigint type in a language that doesn't have it or implement some sorting/searching algorithms or do something that requires concurrency)

Problem:

Proposed by Teo collin

The prisoner's dilemma is a scenario where two people have been arrested and are separated for questioning. Separately and simultaneously, they are asked to betray the other in exchange for a better deal. They can choose to stay silent or talk. If they both stay silent, they will both be convicted on some minor crime and serve for a small amount of time. If either chooses to give up the other, the betrayed is convicted and serves for a considerably longer time while the other goes free. If both choose to betray, to defect, then they both serve the longer sentence.

The prisoner's dilemma is used in game theory/economics and evolutionary biology. You can read more about it here

Now, this game is really boring if you only play it once. The game is considerably more interesting if you play it many times against the same person. Experimenters have done this with real people and when that became boring, people got the idea to design algorithms to play against each other in what were called iterated prisoner's dilemma competitions. People submitted their algorithms and they played 1000 or so matches against every other algorithm. The prison sentences above were translated into numerical scores and algorithms were given a cumulative score for all the games they played against all AI's. This is still going on in certain parts of the net.

The goal of this problem is to create:

  • First, a framework for having AI's compete against each other and keep track of scores. Basically, a system for running the competition
  • Second, several default bots: Tit for tat, tat for tit, defect bot, and cooperate bot.
  • Third, some of our own algorithms for playing the game. What they are allowed to do will be based off the framework.

The framework should come first so I will quickly layout the specifications:

  • Keep track of which algorithms have played, wins/losses, and their scores. The numerical values for the 4 possible scenarios can be chosen at your whim as long as they fulfill the requirements of the game.
  • Inform the algorithms of which algorithms they are competing against.
  • Figure out which AI wins.
  • Provide a transcript of every game played.

After there is some framework, anyone can submit an AI. It would be really quite nice if we were able to actually have a competition.

Notes: I opted for yet another style of documentation. This time I gave a broad but not too specific overview in my write up, but put much more effort into to commenting the code. In general, the more well you comment your code, the better it serves as its own documentation. In this case, almost every single function has a docstring (which is generally a good idea) and I say somewhere in the code what each variable contains. I think the code is sufficiently commented that if you read the overview and then read the code, you should be able to follow the code with relative ease. The only thing that is getting in the way of this is that I didn't refer to any function names in the overview, which probably would have made things clearer.

If you'd like, you can opt to focus more on commenting the code up instead of writing up what every function does outside the code. You can include a high level overview that explain the overall approach, but leave the working details to the comments. This is still probably closer to my typical approach than the approach I just described so you can find some places between the two.

A good place to start is to make sure that almost all functions have a comment that clarifies any uncertainty in their purposes, arguments, return values, and the significance of the return value.

Solution:

Framework proposed by Teo Collin

I have completed the framework for an IPDC. It is written in python 3 and so all the AIs should be written in python 3.

The code makes heavy use of classes. There is a history class. There is a player class. Every player class is created with a function that determines the move for the player. You can see several examples in the code. This move determining function will be given the history object for that function, the opponent, and the current round. The history object for the function keeps historical information about the AI in a format that I hope will be useful.

A history class has a list of previous opponents and a set of dictionaries. The dictionaries map names of opponents to records about the game. Specifically, names of opponents, to move made against them, moves made by them, points won or lost in rounds with them. Mechanisms exist to update the history as the game progresses in the player, history, and game class.

The game class takes the payoffs for the game and allows you to play a game between any two AIs. The class handles the updating of the history objects in order to record the outcomes of the game. Additionally, the player class has a function to communicate with the game class by choices after its choice function is called with the relevant arguments and receiving the results that allow it to update the history function.

All of this machinery comes together so that a round works like this: Each player is given the name of their opponent. The history, the round, and the opponents name are passed to the move determining function of each player. They look up the name in their history classes. Using the information provided in the history class, they make a choice. The result is given to the game class and the pay offs are computed. The payoffs and both players moves are given back to the player. The player updates their score and the history class.

Now, the player class has a list of all players.

All of this machinery comes together in the simulate function to run all existing AIs in all possible combinations. It then uses the history objects for each AI to retrieve information about the net scores of all AIs across all games and the scores of individual one on one competition.

Now, that this is done, someone can write additional AIs to compet or, modify the starting framework in someway to: alter the game conditions (e.g. payoffs) or gather additional information about the results of the game.

If one wants to simply contribute some AIs, simply define the function that would be passed to the player class and post that along with results gathered by running the whole thing.

Here is the code:

import itertools
import random


class History():
    """
    A class for keeping track of a AI's history. It has the name, the list of opponents, the total number of rounds
    and a dictionary where opponents are keys, and either a list of their moves, your moves, or the scores are the items
    """
    def __init__(self, name):
        self.name = name #function name
        self.oppmoves = {}
        self.mymoves = {}
        self.scores = {}
        self.round = 0
        self.listOfOpps = []

    def add_encounter(self, player, myplay, theirplay, result):
        """
        Updates a dictionary after a move. player is the opponent's name, myplay/their play are bools, and result is a score (int).
        """
        if player not in self.listOfOpps:
            self.oppmoves[player] = []
            self.mymoves[player] = []
            self.scores[player] = []
            self.listOfOpps.append(player)

        self.mymoves[player].append(myplay)
        self.oppmoves[player].append(theirplay)
        self.scores[player].append(result)

class Player():
    list_of_players = []
    def __init__(self, movedet):
        """
        :param movedet The function for making a move. It should return True for cooperate and False for Defect. More info in the move function.
        """
        self.score = 0
        self.history = History(repr(movedet.__name__))
        self.move = movedet
        self.name = repr(movedet.__name__) # the player's name is the name of the function!
        Player.list_of_players.append(self)
    def move(self, history, opp, round ):
        """
        :param history - the algorithm's history
        :param The name of the opponent
        :param the round number
        :return True for cooperate or False for defect
        Passes the necessary information to the movedet function, which returns the move
        """
        move = self.movedet(history, opp, round)
        return move
    def update(self, opp, mymove, theirmove, result):
        """
        :param: opp the opposing player's name - needed to look them up
        :param: mymove Bool of my move
        :param: theirmove  bool for their move
        :result: the score
        """
        self.history.add_encounter(opp, mymove, theirmove, result)
        self.score += result


class Game():
    def __init__(self, cop, copd, defc, defect, rounds):
        """
        :param cop: The reward if both cooperate
        :param copd: The reward for the player that copperates when the other defects
        :param defc: The reward for the player that defects when the other cooperates
        :param defect: The reward if they both defect
        :param rounds: The number of rounds to be played
        """
        self.cop = cop
        self.copd = copd
        self.defc = defc
        self.defect = defect
        self.rounds = rounds
    def pay_off(self, moveA, moveB):
        """
        :param moveA Player A's move
        :param moveB Player B's Move
        :return a tuple of (player A's reward, Player B's reward
        """
        if moveA is True and moveB is True:
            return (self.cop, self.cop)
        if moveA is False and moveB is False:
            return (self.defect, self.defect)
        if moveA is True and moveB is False:
            return (self.copd,self.defc)
        if moveA is False and moveB is True:
            return (self.defc,self.copd)

    def play(self, pa, pb):
        """
        :param pa Player A
        :param pb Player B
        Plays the game for the specified number of rounds and updates all relevant information
        """
        x = 0
        while x < self.rounds:
            a = pa.move(pa.history, pb.name, x) #get pa's move
            b = pb.move(pb.history, pa.name, x) #get pb's move
            results = self.pay_off(a, b) #get the pays offs
            pa.update(pb.name, a, b, results[0]) #give each player their pay off and the opponents move
            pb.update(pa.name, b, a, results[1])
            x += 1

# Example functions:
#a - history, b- oppname, c- round
def cooperate(a, b, c):
    return True

def defect(a, b, c):
    return False

def tit_for_tat(a, b, c):
    if c is 0:
        return True
    else:
        return a.oppmoves[b][-1]

def tat_for_tit(a, b, c):
    if c is 0:
        return False
    else:
        return a.oppmoves[b][-1]

def ran_bot(a,b,c):
    test = random.randint(0,1)
    if test is 1:
        return True
    if test is 0:
        return False

# Plays the game with the defined pay offs and prints some information to demonstrate the history
def simulate():
    game = Game(20, -10, 10, 0, 10)
    combinations = list(itertools.combinations(Player.list_of_players,r=2)) # all possible combinations of 2 players
    for x in combinations:
        game.play(x[0], x[1])
    for x in Player.list_of_players:
        print('Player: {0} Score: {1}'.format(x.name, x.score))
    for x in Player.list_of_players:
        for y in Player.list_of_players:
            if y is not x:
                score = sum(x.history.scores[y.name])
                print('{0} scored {1} with {2}'.format(x.name, score, y.name))
