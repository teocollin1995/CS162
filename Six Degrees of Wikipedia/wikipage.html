Six Degrees of Wikipedia

Proposed by Sean Pierre.

Create a program that will take 2 Wikipedia articles (their URLs, not the entire article) as input and return the shortest length between them using only links that are available on the page of the source article and all articles that are traversed to arrive at the destination article.

We define "length" between articles to be the least number of links necessary to click on in order to go from the source article to the destination article.

Example:

Assume that program is named 'sixdegrees' and we wish to go from the Wikipedia article 'Haskell (programming language)' to 'C (programming language)'. We would input to the terminal the following:

sixdegrees http://en.wikipedia.org/wiki/Haskell_(programming_language) http://en.wikipedia.org/wiki/C_(programming_language)
   
[$[Get Code]]

The program should output 2, because we are required to go from

'Haskell (programming language)' -> 'Programming language' -> 'C (programming language)'
[$[Get Code]]

in order to traverse from the Haskell page to the C page (as far as I know, there was no direct link on the Haskell page to the C page)

Note that you may choose another kind of wiki besides Wikipedia if necessary. Let's assume that we are only dealing with English wikipedia articles.

Solution by Jacob Burroughs:

First, a few notes.

  • To invoke my version of sixdegrees, don't include the whole url. You can write
sixdegrees Haskell_(programming_language) C_(programming_language)
   
[$[Get Code]]

or

sixdegrees "Haskell (programming language)" "C (programming language)"
   
[$[Get Code]]
  • I originally wrote a version that ran directly against the wikipedia server; it was awfully slow (many cases would probably have run for hours/days. Therefore, in order to use this script, you must load the {wikiname}-latest-page.sql.gz and {wikiname}-latest-pagelinks.sql.gz into a sql server.
  • Currently, I have only run it against Simple English Wikipedia (simple.wikipedia.org), but I'm working on getting it running with the regular English Wikipdia.

Code can be found on github http://git.io/KtUdaQ A web wrapper can be accessed at http://maths22.com/cs/simpledegrees.php

Proposed by Teo Collin

This solution uses a breadth first search algorithm written in python. It uses a the queue data structure to keep track of pages that need to be looked. It uses networkx to keep track of the overall structure of the tree.

It is run like this:

python3.4 main.py '/wiki/Haskell_(programming_language)' '/wiki/C_(programming_language)'
[$[Get Code]]

And, as with the solution above, it runs extremely slowly when running against wikipedia servers, but has no alternative yet. I plan to write a program with converts wikipedia's page link file into some usable graph object for fast searching. This will probably be a separate solution.

Here is the solution that runs against wikipedia's servers:

import networkx as nx
import sys
import requests
from bs4 import BeautifulSoup
from collections import deque

toSort = True

wikgraph = nx.DiGraph()

arg1 = sys.argv[1]
arg2 = sys.argv[2]


def page_eval_sort(links):
    look_for = ((arg2.replace('/wiki/','').replace('(','').replace(')','')).split('_')) + ['list' ]
    pre_sort = [[0,x] for x in links]
    for x in look_for:
        for y in pre_sort:
            if x in y[1]:
                y[0] = y[0] + 1

    pre_sort.sort(key=lambda a: a[0])
    return [x[1] for x in pre_sort]


def get_wiki_page_soup(old):
    new = 'https://en.wikipedia.org' + old
    print(new)
    r = requests.get(new)
    print(r)
    if r.status_code == 404:
        print(old + " 404'd")
        return()
    return BeautifulSoup(r.text)


def get_all_page_urls(soup):
    a = soup.find_all('a')
    rawa = [x.get('href') for x in a]
    rawn = [x for x in rawa if x is not None and 'org' not in x and '/wiki/' in x and ':' not in x and 'Main_Page' not in x]
    return rawn


def tree_append(page):
    print("page: " + page)
    soup = get_wiki_page_soup(page)
    newpages = get_all_page_urls(soup)
    print(newpages)
    wikgraph.add_edges_from([(page, x) for x in newpages if x not in wikgraph.nodes_iter()])
    if toSort:
        return page_eval_sort(newpages)
    return newpages


def tree_construct_search_breadth_limited(page1, page2):
    que = deque([page1])
    while que:
        considering = que.pop()
        print(considering)
        new = tree_append(considering)
        if page2 in new:
            print("Done!")
            print("Path:")
            print(nx.shortest_path(wikgraph, arg1, arg2))
            sys.exit(0)
        print("Extending")
        que.extendleft(new)


tree_construct_search_breadth_limited(arg1, arg2)
