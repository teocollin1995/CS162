SunWooPark /

RandomConundrumSolutions

proposed by Sun Woo Park

In the quiz program "Countdown" on Channel 4, UK, the contestants choose 9 random alphabets and create a word using the letters. The specific rules are in this page: http://en.wikipedia.org/wiki/Countdown_%28game_show%29#Letters_round. The lexicographers / celebrities in the "dictionary corner" determine what are the possible solutions for the given random set of alphabets. Instead of the lexicographers, let's have the computers do the job. Make a program which gives all possible words (which is comprised of more than 4 alphabets) generated from a set of chosen letters by the user. If there aren't any possible words, the program may print out an error message.

A possible solution for 5 random alphabets may look like this:

$ conundrum axtes
taxes
east
seat
teas
eats
sate
saxe
sext
axes

$ conundrum xqvyl
No Solution

The solutions may differ based on which dictionary you are using for the code.

Solution by Teodoro Fields Collin:

First pass created using Python 2.7.4.

import itertools
from sys import argv


def cands(inputs):
    """
    Finds all permutations with more than 4 characters of a list of characters
    :arg List of characters
    :return list of strings
    """
    # The below could probably be simplified a bit....
    return map(''.join, list(itertools.chain.from_iterable([ map (list, (itertools.permutations(inputs, x))) for x in range(4, len(inputs)+1)])))

def dict_make():
    """
    Creates a dictionary of all words greater than length 4
    :args None
    :return A dictionary
    """
    f = open('words.txt','r')
    dict1 = f.read().splitlines()
    return filter(lambda z:len(z) >= 4, dict1)

def main():
    """
    Solves a conundrum as specified here: http://brick.cs.uchicago.edu/Courses/CMSC-16200/2015/pmwiki/pmwiki.php/SunWooPark/RandomConundrumSolutions
    :arg None
    :return None
    """
    string = argv[1] # The 2nd command line argument.
    possible = cands(string)
    dictionary = dict_make()
    results = [possible[x] for x in xrange(0, len(possible)) if possible[x] in dictionary]
    if len(results) == 0:
        print "No Solution"
        return ()
    else:
        for x in range(0, len(results)):
            print results[x]
    return ()

main()

Itertools is a very useful python library that has a permutations function and other useful list operations. Using their permutations function, the list function, and some other things, all permutations with length 4 or greater are created. A dictionary is created by reading in my the dictionary located in /usr/share/dict and splitting it by lines. Since we only needs words of length 4 or greater, I filtered it by length. Then, simply filter out all the strings that aren't in the dictionary, and output them.

----:~----$ python main.pyc xqvyl
No Solution

Solution by Bowen Wang

It's similar to the previous solution, but implemented in haskell. There are some very useful functions for this problem in Data.List.

import Data.List
import System.IO
import Control.Monad

generate :: String -> [String]
generate xs = filter (\x -> length x >= 4) (nub $ concat $ map subsequences (permutations xs))

mainloop inh list = do
                    ineof <- hIsEOF inh
                    if ineof
                    then return list
                    else do
                       inpStr <-hGetLine inh
                       let list1 =  inpStr : list
                       mainloop inh list1

main :: IO ()
main = do
       inh <- openFile "words.txt" ReadWriteMode
       wordDict <- mainloop inh []
       letters <- getLine
       case intersect (generate letters) wordDict of
         [] -> putStrLn "No Solution"
         _  -> mapM_ putStrLn (intersect (generate letters) wordDict)
