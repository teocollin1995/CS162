FourIsTheMagicNumber

There is a cool property of how numbers are written in english; this is often called "Four Is The Magic Number".

The idea is there is some function that can map from number to number recursively, though once it hits 4, it is done. The function takes in a number, represents its spelled-out name, and then tallies the letters and recurses with the number of letters.

For instance, look at 7:

f(7) = length(seven) = 5
f(5) = length(five) = 4
f(4) = length(four) = 4 (and 4 is the magic number).

Write code that prompts the user for a number (x), and then takes x, recurses down, printing at each step what f(x) is. One should find that 4 is indeed the magic number.

Some questions / standards to consider:

How do you represent numbers such as 25 or 147 ("twenty-five", or "TwentyFive"; "one-hundred-and-forty-seven" or "one-hundred-forty-seven" or "OneHundredFortySeven")?

(It shouldn't make much of a difference, considering that 4 is the only thing that maps to itself and everything else maps to something that is only a finite number of steps from 4)

The biggest issue in the problem is how to actually map strings from standard input into their own letter representations; undoubtedly there are online algorithms that do such manipulation (think how many programs must utilize this!) but try to come up with your own.

Contributed by SunWooPark

The following code can deal with digits with at most 150 zeroes. (We can add more if we want to by changing the function for fname). This code was implemented in Haskell. It seems as if the code could be abbreviated much more but this is the best I could think of.

module FourMagic where

import Data.List

-- The main function trans transforms a number to its spelling.
trans :: String -> String
trans "0" = "Zero"
trans "1" = "One"
trans "2" = "Two"
trans "3" = "Three"
trans "4" = "Four"
trans "5" = "Five"
trans "6" = "Six"
trans "7" = "Seven"
trans "8" = "Eight"
trans "9" = "Nine"
trans "10" = "Ten"
trans "11" = "Eleven"
trans "12" = "Twelve"
trans "13" = "Thirteen"
trans "14" = "Fourteen"
trans "15" = "Fifteen"
trans "16" = "Sixteen"
trans "17" = "Seventeen"
trans "18" = "Eighteen"
trans "19" = "Nineteen"
trans "20" = "Twenty"
trans "30" = "Thirty"
trans "40" = "Fourty"
trans "50" = "Fifty"
trans "60" = "Sixty"
trans "70" = "Seventy"
trans "80" = "Eighty"
trans "90" = "Ninety"
trans a = subtrans (subdiv (reverse a)) where
   subdiv :: String -> [String]
   naming :: String -> String
   subtrans :: [String] -> String
   isNempty :: String -> Bool
   fname :: [String] -> String
   ffname :: Int -> String
-- divides the digits to a list of strings, each string of 3 digits
   subdiv [] = []
   subdiv l = (reverse (take 3 l)):(subdiv (drop 3 l))
-- transforms the list of strings into literal spelling of a number
   subtrans p = fname (reverse (map naming p))
-- spells the number for each string of 3 digits
   naming [] = ""
   naming [a]
      | a == '0' = ""
      | otherwise = trans [a]
   naming [a,b]
      | a == '0' && b == '0' = ""
      | a == '1' = trans ([a] ++ [b])
      | a == '0' = trans [b]
      | otherwise = trans ([a] ++ "0") ++ trans [b]
   naming [a,b,c]
      | a == '0' && b == '0' && c == '0' = ""
      | a == '0' && b == '0' = trans [c]
      | a == '0' && b == '1' = trans ([b] ++ [c])
      | a == '0' && c == '0' = trans ([b] ++ "0")
      | b == '0' && c == '0' = trans [a] ++ "Hundred"
      | b == '0' = trans [a] ++ "Hundred" ++ trans [c]
      | b == '1' = trans [a] ++ "Hundred" ++ trans ([b] ++ [c])
      | c == '0' = trans [a] ++ "Hundred" ++ trans ([b] ++ "0")
      | otherwise = trans [a] ++ "Hundred" ++ trans ([b] ++ "0") ++ trans [c]
-- Checks whether the length of string is not zero
   isNempty a = length a /= 0
-- spells specific digits (thousands, millions, etc) for each string of 3 digits
   fname [] = []
   fname (a:as)
      | length (a:as) == 0 = ""
      | length (a:as) == 1 = a
      | length (a:as) == 2 && isNempty a = a ++ "Thousand" ++ fname as
      | length (a:as) == 3 && isNempty a = a ++ "Million" ++ fname as
      | length (a:as) == 4 && isNempty a = a ++ "Billion" ++ fname as
      | length (a:as) == 5 && isNempty a = a ++ "Trillion" ++ fname as
      | length (a:as) == 6 && isNempty a = a ++ "Quadrillion" ++ fname as
      | length (a:as) == 7 && isNempty a = a ++ "Quintillion" ++ fname as
      | length (a:as) == 8 && isNempty a = a ++ "Sextillion" ++ fname as
      | length (a:as) == 9 && isNempty a = a ++ "Septillion" ++ fname as
      | length (a:as) == 10 && isNempty a = a ++ "Octillion" ++ fname as
      | length (a:as) == 11 && isNempty a = a ++ "Nonillion" ++ fname as
      | length (a:as) >= 12 && length (a:as) <= 21 && isNempty a = a ++ ffname (length (a:as) - 12) ++ "Decillion" ++ fname as
      | length (a:as) >= 22 && length (a:as) <= 31 && isNempty a = a ++ ffname (length (a:as) - 22) ++ "Vigintillion" ++ fname as
      | length (a:as) >= 32 && length (a:as) <= 41 && isNempty a = a ++ ffname (length (a:as) - 32) ++ "Trigintillion" ++ fname as
      | length (a:as) >= 42 && length (a:as) <= 51 && isNempty a = a ++ ffname (length (a:as) - 42) ++ "Quadragintillion" ++ fname as
      | length (a:as) >= 52 = "Exit"
      | otherwise = a ++ fname as
-- deals with naming digits such that the numbers have more than 30 zeroes.
   ffname p
      | p `mod` 10 == 1 = "Un"
      | p `mod` 10 == 2 = "Duo"
      | p `mod` 10 == 3 = "Tre"
      | p `mod` 10 == 4 = "Quattuor"
      | p `mod` 10 == 5 = "Quin"
      | p `mod` 10 == 6 = "Sex"
      | p `mod` 10 == 7 = "Septen"
      | p `mod` 10 == 8 = "Octo"
      | p `mod` 10 == 9 = "Novem"
      | otherwise = ""

-- changes the length of a string to string
change :: String -> String
change a = show.length $ a

-- Repeating Function trans
func :: String -> IO ()
func z = do
   if z == "4"
   then
      putStrLn "f(4) = length(four) = 4 (and 4 is the magic number)"
   else do
      putStrLn ("f(" ++ z ++ ") = length(" ++ trans z ++ ") = " ++ (change (trans z)))
      func (change (trans z))

-- Main IO
fourmagic :: IO ()
fourmagic = do
   x <- getLine
   func x

Solution in Python 3 by Teo Collin

The problem of converting integers to words and establishing a consistent standard has already been solved by whoever wrote the python library Inflect. The desire not to reinvent the wheel reduces this problem to a while loop:

from sys import argv
import inflect

eng = inflect.engine()
current_word = eng.number_to_words(argv[1])
n = 0
temp = 0
while temp != 4:
    temp = len(current_word)
    print("We are at depth {0}\n{1} is the current word with a length of {2}".format(n, current_word, temp))
    current_word = eng.number_to_words(temp)
    n += 1
print("We have reached four at a depth of {0}".format(n))
exit(0)
